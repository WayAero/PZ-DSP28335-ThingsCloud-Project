# DSP28335 + ESP32 ThingsCloud 使用说明

更新时间：2026-05-15

## 1. 项目组成

本项目由三部分组成：

1. DSP28335 开发板：负责 ADC、电机、继电器、蜂鸣器、LED、按键、LCD9648 显示。
2. ESP32：负责 Wi-Fi、DHT22 温湿度、ThingsCloud MQTT、DSP 串口桥接。
3. ThingsCloud：负责设备属性显示和属性下发控制。

工程路径：

```text
DSP 工程：
<DSP工程路径>

ESP32 工程：
<ESP32工程路径>
```

## 2. 接线说明

### DSP 与 ESP32 串口

```text
DSP GPIO35 / SCITXDA -> ESP32 GPIO16 / RX2
DSP GPIO36 / SCIRXDA <- ESP32 GPIO17 / TX2
DSP GND              <-> ESP32 GND
```

注意：

- ESP32 不使用 RX0/TX0 连接 DSP。
- RX0/TX0 留给 ESP32 的 USB 日志串口。
- DSP 和 ESP32 必须共地。
- DSP 串口参数为 `9600 8N1`。

### 电机

电机使用 TC1508S 第二通道：

```text
DSP GPIO4 / ePWM3A -> TC1508S INC
DSP GPIO5 / ePWM3B -> TC1508S IND
```

### 蜂鸣器

```text
DSP GPIO6 / ePWM4A -> 蜂鸣器
```

### LCD9648

```text
GPIO0  -> SCL
GPIO60 -> SDA
GPIO1  -> RS
GPIO2  -> CS
GPIO3  -> RSET
```

LCD9648 当前显示 ADC 电压。

## 3. 烧录和启动

### DSP

使用 CCS 7 打开 DSP 工程：

```text
<DSP工程路径>
```

当前已生成的 DSP 输出文件：

```text
<DSP工程路径>\Debug\DSP2833x_Course Project.out
```

下载到 DSP 后运行程序。

DSP 启动后，串口会先输出：

```json
{"boot":1}
```

随后每秒输出一次完整状态：

```json
{"led":0,"beep":0,"relay":0,"motor_enable":0,"motor_dir":1,"motor_speed":0,"adc_mv":2200,"alarm":0}
```

### ESP32

使用 PlatformIO 打开 ESP32 工程：

```text
<ESP32工程路径>
```

烧录后打开 ESP32 串口监视器。

串口监视器参数：

```text
115200
```

正常情况下可以看到类似日志：

```text
DSP -> ESP32: {"led":0,"beep":0,"relay":0,"motor_enable":0,"motor_dir":1,"motor_speed":0,"adc_mv":2200,"alarm":0}
```

ThingsCloud 属性下发后，可以看到类似日志：

```text
MQTT >> [attributes/push] {"led":1}
ESP32 -> DSP: {"led":1}
```

## 4. ThingsCloud 属性

平台设备属性中查看这些字段：

```text
temperature
humidity
adc_voltage
led
beep
relay
motor_enable
motor_dir
motor_speed
alarm
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| temperature | DHT22 温度 |
| humidity | DHT22 湿度 |
| adc_voltage | DSP ADC 电压，单位 V |
| led | LED 状态，0 关，1 开 |
| beep | 蜂鸣器实际输出状态，0 停，1 响 |
| relay | 继电器状态，0 关，1 开 |
| motor_enable | 电机使能，0 停，1 转 |
| motor_dir | 电机方向，1 正转，2 反转 |
| motor_speed | 电机档位，0-7 |
| alarm | ADC 报警状态，0 正常，1 报警 |

## 5. ThingsCloud 下发控制

ThingsCloud 属性下发 JSON 使用紧凑格式。

平台下发时不需要在末尾加 `\n`。

ESP32 转发给 DSP 时会自动追加串口换行。

常用下发示例：

```json
{"led":1}
```

```json
{"led":0}
```

```json
{"relay":1}
```

```json
{"beep":1}
```

```json
{"beep":0}
```

```json
{"motor_enable":1}
```

```json
{"motor_speed":7}
```

```json
{"motor_dir":2}
```

```json
{"motor_enable":0}
```

```json
{"alarm_clear":1}
```

也可以一次下发多个字段：

```json
{"motor_enable":1,"motor_speed":5,"motor_dir":1}
```

## 6. DSP 串口命令

如果直接用串口调试助手连接 DSP，发送 JSON 时必须带换行。

示例：

```json
{"led":1}
```

发送设置：

```text
9600 8N1
发送新行 / 回车换行：开启
```

如果使用 HEX 发送，需要在末尾追加：

```text
0A
```

原因：

- DSP 使用 JSON 行协议。
- 没有收到 `\n` 时，DSP 会继续缓存。
- 收到 `\n` 后才会解析并执行命令。

## 7. 报警规则

ADC 报警阈值：

```text
adc_mv >= 2500：进入报警
adc_mv <= 2400：自动解除报警
2400 < adc_mv < 2500：保持原状态
```

报警时现象：

- `alarm=1`
- `beep=1`
- LED2 亮
- 蜂鸣器响

解除或暂停报警：

```text
KEY9：暂停报警约 10 秒
alarm_clear=1：暂停报警约 10 秒
```

如果 10 秒后 ADC 仍高于 2500mV，会再次报警。

如果 ADC 降到 2400mV 以下，报警自动清零。

## 9. 电机控制

电机字段：

```text
motor_enable: 0/1
motor_dir: 1 正转，2 反转
motor_speed: 0-7
```

推荐测试顺序：

1. 下发 `{"motor_enable":1}`。
2. 下发 `{"motor_speed":7}`。
3. 确认电机启动。
4. 下发 `{"motor_speed":3}`。
5. 确认电机转速变化。
6. 下发 `{"motor_dir":2}`。
7. 确认电机方向变化。
8. 下发 `{"motor_enable":0}`。
9. 确认电机停止。

已知现象：

- 部分电机初始 1-6 档可能不转。
- 7 档启动后，后续低档通常可以正常转。
- 这更像电机静摩擦或启动扭矩问题。

### 单板测试

1. DSP 下载并运行最新 `.out`。
2. 串口确认先收到 `{"boot":1}`。
3. 串口确认每秒收到完整 JSON。
4. 调节 ADC，确认 `adc_mv` 变化。
5. 确认 LCD9648 显示 ADC 电压。
6. 发送 `{"led":1}`，确认 LED 动作。
7. 发送 `{"relay":1}`，确认继电器动作。
8. 发送 `{"beep":1}`，确认蜂鸣器响。
9. 发送 `{"beep":0}`，确认蜂鸣器停。
10. 发送电机控制命令，确认电机动作。
11. ADC 调到 2500mV 以上，确认报警。
12. 发送 `{"alarm_clear":1}`，确认报警暂停约 10 秒。

### 云端闭环测试

1. DSP 和 ESP32 按正确串口接线连接。
2. DSP 上电运行。
3. ESP32 上电运行。
4. ESP32 串口监视器确认收到 `DSP -> ESP32`。
5. ThingsCloud 确认温湿度更新。
6. ThingsCloud 确认 `adc_voltage` 更新。
7. ThingsCloud 下发 `{"led":1}`。
8. 确认 ESP32 日志出现 `ESP32 -> DSP`。

## 12. 常见问题

### 只能收到 `{"boot":1}`

检查：

- DSP 是否下载的是最新 `.out`。
- 程序是否已经运行。
- ADC 是否阻塞。
- Timer0 轮询 tick 是否正常。

当前工程中这些问题已修复。

### ESP32 收不到 DSP JSON

检查接线：

```text
DSP TX GPIO35 -> ESP32 GPIO16
DSP RX GPIO36 <- ESP32 GPIO17
GND           <-> GND
```

不要接到 ESP32 RX0/TX0。

### ThingsCloud 下发后 DSP 没动作

检查：

- ESP32 串口日志是否出现 `MQTT >> [attributes/push]`。
- ESP32 串口日志是否出现 `ESP32 -> DSP`。
- DSP 和 ESP32 是否共地。
- DSP 串口是否接到 ESP32 GPIO16/GPIO17。

### `alarm_clear=1` 后又马上报警

当前版本已修复。

正常现象：

- 下发 `{"alarm_clear":1}` 后报警暂停约 10 秒。
- 10 秒后 ADC 仍高于 2500mV，会再次报警。

### `beep=1` 后 `alarm_clear` 不能让蜂鸣器停

这是正常现象。

`beep=1` 是手动蜂鸣器开关。

`alarm_clear=1` 只暂停报警，不关闭手动蜂鸣器。

需要关闭手动蜂鸣器时，下发：

```json
{"beep":0}
```
